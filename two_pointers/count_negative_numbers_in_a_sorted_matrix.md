# Count Negative Numbers in a Sorted Matrix

**난이도:** Easy  
**문제 링크:** [LeetCode - Count Negative Numbers in a Sorted Matrix](https://leetcode.com/problems/count-negative-numbers-in-a-sorted-matrix/)

## 문제 설명

`m x n` 크기의 그리드 `grid`가 주어집니다. 그리드는 **행과 열 모두 내림차순**으로 정렬되어 있습니다.

그리드에 있는 **음수의 개수**를 반환하세요.

## 제약 조건

- `m == grid.length`
- `n == grid[i].length`
- `1 <= m, n <= 100`
- `-100 <= grid[i][j] <= 100`

## 예제

### 예제 1
**입력:** `grid = [[4,3,2,-1],[3,2,1,-1],[1,1,-1,-2],[-1,-1,-2,-3]]`  
**출력:** `8`  
**설명:** 그리드에 8개의 음수가 있습니다.

### 예제 2
**입력:** `grid = [[3,2],[1,0]]`  
**출력:** `0`  
**설명:** 그리드에 음수가 없습니다.

## 솔루션

### 코드
```typescript
/**
 * @param {number[][]} grid
 * @return {number}
 */
function countNegatives(grid: number[][]): number {
    const m = grid.length;
    const n = grid[0].length;

    let answer = 0;
    let i = 0;      // 행 포인터
    let j = n - 1;  // 열 포인터 (오른쪽 끝에서 시작)

    // 오른쪽 위 모서리에서 시작하여 탐색
    while (i < m && j >= 0) {
        if (grid[i][j] < 0) {
            // 현재 위치가 음수면, 이 열의 아래 모든 행도 음수
            answer += (m - i);
            j--; // 왼쪽으로 이동
        } else {
            // 양수면 다음 행으로 이동
            i++;
        }
    }

    return answer;
}
```

### 접근 방법

**두 포인터(Two Pointers)** 기법을 사용합니다.

1. **시작 위치**: 오른쪽 위 모서리 `(0, n-1)`에서 시작합니다.
2. **탐색 전략**:
   - 현재 위치가 **음수**인 경우:
     - 그리드가 내림차순 정렬되어 있으므로, 현재 열의 아래 모든 행(`m - i`개)도 음수입니다.
     - 음수 개수에 `(m - i)`를 더하고, 열 포인터를 왼쪽으로 이동합니다 (`j--`).
   - 현재 위치가 **양수 또는 0**인 경우:
     - 현재 행의 왼쪽은 모두 양수이므로, 행 포인터를 아래로 이동합니다 (`i++`).
3. **종료 조건**: 행 포인터가 `m`에 도달하거나 열 포인터가 `-1`에 도달하면 종료합니다.

**핵심 아이디어:** 
- 그리드가 행과 열 모두 내림차순으로 정렬되어 있으므로, 한 위치가 음수면 그 아래와 왼쪽은 모두 음수입니다.
- 오른쪽 위에서 시작하여 음수를 찾으면 해당 열의 아래 모든 행을 한 번에 카운트할 수 있습니다.

### 시간 복잡도
**`O(m + n)`**

- 최악의 경우 행 포인터가 `m`번, 열 포인터가 `n`번 이동하므로 `O(m + n)`입니다.
- 각 위치를 한 번씩만 방문하므로 선형 시간입니다.

### 공간 복잡도
**`O(1)`**

- 추가 공간 없이 포인터 변수만 사용하므로 `O(1)`입니다.
